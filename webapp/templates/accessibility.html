Accessibility Checks


While developing the frontend of the web application, I aimed to keep responsiveness and accessibility in mind. I referred to W3C's accessibility resources and checked my code against this list as detailed below.

Page Titles


I ensured that each page has a title that adequately distinguishes the page from other pages.

Image Text Alternatives


After adding images to web pages, I ensured that each image had appropriate and functional alt text for those who cannot see the image.

Headings


While developing each page, I was not adding headings but using bold text as a substitute instead. Later I replaced the bold text accordingly with headings.

Contrast Ratio


Initially, each page was using a gray text but later on I changed all text to use black text for better contrast on a white background.

Resize Text


When zooming in on pages, the text size increases as expected without any issues.

Keyboard Access and Visual Focus


On the credit card approval form, keyboard navigation is supported. Links on pages are also highlighted appropriately when hovered over.

Forms, Labels, and Errors


Every form control has a label associated with it, and labels are positioned correctly. Error handling is supported when incorrect inputs are entered and also if inputs are left empty.
